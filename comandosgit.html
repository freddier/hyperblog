Comandos mas comunes en Git y Git Hub

Generales

pwd Muestra la car´peta donde estamos

~ Nos lleva directo al Home de la carpeta cuenta del usuario

ls Muestra el contenido de la carpeta actual

ls -al Muestra todos los archivos de la carpeta actual en una lista

clear Limpiar la pantalla de la terminal

cd mnt/Disco Para cambiar a otro disco (funciona en linux)

cd letra Muestra las carpetas o archivos que empizan por la letra especificada

mkdir Crea una carpeta/directorio

touch nombre_archivo.extension Crea un archivo vacio con la extension especificada

history Muestra una lista numerada de todos los comandos utilizados recientemente en la terminal (otra forma de ejecutar el comando es colocando !numero_del_comando_en_la_lista)

rm nombre_archivo Elimina el archivo especificado (Este comando es delicado y existen distintos tipos)

cat nombre_archivo Muestra directamente en la terminal el contenido del archivo especificado

comando --help Muestra la ayuda sobre el comando especificado

Inicializar Git

git init Inicializa Git en la carpeta donde estamos ubicados

git config Muestra todas las configuraciones que tiene Git

git config -l Muestra la configuracion por defecto especifica de mi Git (tambien se puede usar git config --list)

git config -l --show-origin Muestra donde estan guardadas en windows las configuracios de Git

git config --global user.name "nombre_usuario" Cambia el nombre de usuario de Git (Generalmente se maneja un usuario por PC). Para cambiar el mail es lo mismo pero usando user.email

Para Agregar

git add nombre_archivo Agrega a Staging los cambios realizados al archivo especificado que esta en la carpeta actual de trabajo (directorio en windows) si se usa git add . agrega todos los cambios del proyecto

git commit -m "mensaje_del_commit" Agrega a Git repository los cambios con el mensaje especificado (si entra al editor Vim, salir con Esc + w + q tambien con Esc + z +z)

git commit -am "mensaje_del_commit" Une los dos comandos anteriores en uno, es decir hace un git add y un git commit al mismo tiempo (solo funciona si al archivo/proyecto ya se le ha hecho un git add antes)

git checkout ID_commit nombre_archivo Regresa a Staging una version anterior del archivo (git checkout master nombre_archivo regresa a la version mas nueva que está en el master, luego hay que hacer git commit)

Para Mostrar

git status Muestra el estado del proyecto en relacion a los ultimos cambios que se icieron en el mismo

git show nombre_archivo Muestra el detalle del ultimo cambio que se le hizo al archivo especificado en base al ultimo commit (si se coloca git show solo, muestra de todo el proyecto)

git diff commitA commitB Muestra la diferencia especifica en los cambios de los dos commit especificados (se recomienda colocar la version mas vieja primero para ver de mejor forma la diferencia)

git log nombre_archivo Muestra los ID de los commit realizados, es decir muestra la historia del archivo (si se coloca git log solo, muestra todos los commit del proyecto)

git log --stat Muestra los ID de los commit realizados con el detalle de los cambios especificos en cada commit

git log --all Muestra literalmente todo lo que se ha hecho historicamente en el proyecto

git log --all --graph Muestra todos los cambios con lineas de colores que destacan como han sido los cambios en las ramas

git log --all --graph --decorate --oneline Muestra todos los cambios de manera muy resumida (es muy util para ver la historia resumida)
